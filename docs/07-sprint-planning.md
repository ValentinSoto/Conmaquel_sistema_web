# Sprint Planning

## Descripción

El Sprint Planning es una reunión fundamental dentro de la metodología Scrum, en la cual el equipo de desarrollo, el Product Owner y el Scrum Master definen el trabajo que se realizará durante cada Sprint. Durante esta reunión se seleccionan las historias de usuario con mayor prioridad del Product Backlog y se establecen los objetivos y tareas necesarias para completar el incremento del producto.

Para el proyecto **Sistema Web de Gestión Integral para Conmaquel**, se planificaron tres Sprint que permitieron desarrollar progresivamente las funcionalidades principales del sistema.

---

# Objetivos de los Sprint

## Sprint 1

### Objetivo

Implementar la estructura inicial del sistema y desarrollar los módulos de autenticación, gestión de usuarios y administración de productos.

### Historias de Usuario

- HU-01 Inicio de sesión.
- HU-02 Gestión de usuarios.
- HU-03 Registro de productos.
- HU-04 Actualización de productos.

---

## Sprint 2

### Objetivo

Desarrollar el módulo de reservas y consulta de productos.

### Historias de Usuario

- HU-05 Consulta del catálogo.
- HU-06 Reserva de productos.
- HU-07 Gestión de reservas.

---

## Sprint 3

### Objetivo

Implementar el módulo de ventas, generación de reportes y funcionalidades complementarias.

### Historias de Usuario

- HU-08 Registro de ventas.
- HU-09 Reportes.
- HU-10 Búsqueda avanzada.
