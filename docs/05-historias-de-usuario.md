# Historias de Usuario

## Descripción

Las historias de usuario representan las necesidades y expectativas de los usuarios finales del sistema. Se redactan desde la perspectiva del usuario y permiten definir las funcionalidades que serán desarrolladas durante cada Sprint de la metodología Scrum.

Cada historia de usuario incluye una breve descripción, el beneficio esperado y una prioridad, facilitando la planificación del Product Backlog.

# Criterios de Aceptación

## HU-01 – Inicio de Sesión

- El usuario deberá ingresar su nombre de usuario y contraseña.
- El sistema validará las credenciales.
- Si los datos son correctos, permitirá el acceso.
- Si son incorrectos, mostrará un mensaje de error.

---

## HU-06 – Reserva de Productos

- El cliente podrá seleccionar uno o varios productos.
- El sistema verificará la disponibilidad.
- Se registrará la reserva.
- El usuario recibirá una confirmación.

---

# Beneficios

- Facilitan la comunicación con el cliente.
- Permiten organizar el Product Backlog.
- Ayudan a planificar los Sprint.
- Mejoran el seguimiento del desarrollo.
- Favorecen la entrega incremental de funcionalidades.

---

# Conclusión

Las historias de usuario permiten comprender las necesidades de los diferentes actores del sistema y constituyen la base para la planificación del Product Backlog y el desarrollo iterativo mediante la metodología Scrum.
