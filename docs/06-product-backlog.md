# Product Backlog

## Descripción

El Product Backlog es una lista priorizada de todas las funcionalidades, mejoras y requisitos que forman parte del Sistema Web de Gestión Integral para Conmaquel. Este documento sirve como guía para la planificación y ejecución de los Sprint dentro de la metodología Scrum.

Cada elemento del backlog se representa mediante una Historia de Usuario, la cual ha sido estimada utilizando Story Points y clasificada según su prioridad.

# Criterios de Priorización

Los elementos del Product Backlog fueron priorizados considerando:

- Valor para el negocio.
- Complejidad técnica.
- Dependencias entre funcionalidades.
- Riesgos del proyecto.
- Impacto para los usuarios.

---

# Beneficios del Product Backlog

- Organiza el trabajo del equipo Scrum.
- Facilita la planificación de los Sprint.
- Permite controlar el avance del proyecto.
- Ayuda a identificar prioridades.
- Favorece la entrega incremental de valor.

---

# Relación con los Sprint

El Product Backlog constituye la principal fuente de trabajo para el Sprint Planning. Durante cada Sprint se seleccionan los elementos de mayor prioridad para conformar el Sprint Backlog.

---

# Conclusión

El Product Backlog permitió organizar y priorizar las funcionalidades del Sistema Web de Gestión Integral para Conmaquel, facilitando la planificación del proyecto bajo la metodología Scrum y asegurando una entrega progresiva de valor al cliente.
